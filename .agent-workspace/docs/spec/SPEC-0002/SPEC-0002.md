# Specification: SPEC-0002 - Global Book Catalog

## 1. Metadata
*   **Spec ID:** SPEC-0002
*   **Version:** 1.0.0
*   **Status:** Implemented
*   **Owner:** BA Agent, Architect Agent
*   **Source Milestone:** Functional Alignment
*   **Last Updated:** 2026-07-07

## 2. Scope Definition
*   **Problem Statement:** System must maintain a unique, deduplicated global book catalog to avoid duplicate entries and enable library sharing across users.
*   **Business Outcome:** Single source of truth for books; efficient search; reduces data redundancy.
*   **In-Scope:** Book creation, deduplication by ISBN/title-author, search by title/author/ISBN, basic metadata fields (cover, publisher, pages).
*   **Out-of-Scope:** Book recommendations, advanced filtering (genre hierarchy), book reviews in catalog (reviews per user-book relation only).

## 3. Actors & Preconditions
*   **Primary Actors:** End user, system administrator.
*   **Dependencies:** None (autonomous catalog service).
*   **Preconditions:** Book table initialized, ISBN/title-author indexes created.

## 4. Functional Requirements
*   **REQ-001:** Accept book creation with: title, author(s), ISBN-13, ISBN-10 (optional), publisher, publication_date, pages, language, genres, cover_url.
*   **REQ-002:** Deduplicate by canonical key: ISBN-13 (if present) OR normalized(title + primary_author).
*   **REQ-003:** Search books by title (substring, case-insensitive), author name, ISBN.
*   **REQ-004:** Return book metadata including unique book_id, creation_timestamp, last_modified_timestamp.
*   **REQ-005:** Provide endpoints: POST /books, GET /books/search, GET /books/{id}, PUT /books/{id}.
*   **REQ-006:** Admin can mark book as canonical after duplicate detection.

## 5. Non-Functional Requirements
*   **NFR-001:** Search latency: <500ms for typical queries (1000-book index).
*   **NFR-002:** ISBN validation: reject invalid checksum; warn on missing check digit.
*   **NFR-003:** Title/author normalization: trim whitespace, lowercase, remove accents.

## 6. Business Rules
*   **RULE-001:** ISBN-13 is primary deduplication key; ISBN-10 is secondary validation only.
*   **RULE-002:** If ISBN unavailable, use normalized(title + primary_author) as fallback dedup key.
*   **RULE-003:** Book record is immutable after creation (except cover_url, metadata corrections by admin).
*   **RULE-004:** Each genre is a pre-defined enum; users cannot create custom genres.
*   **RULE-005:** Book created_by tracks uploader user_id for audit; admin can bulk-import with system user ID.

## 7. Acceptance Criteria
*   **AC-001:** Create book with ISBN-13 → second user tries to create same ISBN → system returns existing book_id, no duplicate.
*   **AC-002:** Create book without ISBN → second user creates with same title + author → deduplication triggers, returns existing book_id.
*   **AC-003:** Search "the hobbit" → returns book with exact title and all case variants.
*   **AC-004:** Search by ISBN-10 (when ISBN-13 absent) → finds book successfully.
*   **AC-005:** Admin updates book cover_url → returned in subsequent GET /books/{id} calls within 5s.
