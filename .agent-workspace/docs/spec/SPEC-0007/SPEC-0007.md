# Specification: SPEC-0007 - Unified Book Search with External Integration

## 1. Metadata
*   **Spec ID:** SPEC-0007
*   **Version:** 1.0.0
*   **Status:** In Progress
*   **Owner:** BA Agent, Architect Agent
*   **Source Milestone:** Enhanced Book Discovery
*   **Last Updated:** 2026-07-08
*   **Implementation Status:** Backend Implemented; Web Frontend Integrated; final runtime/E2E validation pending

## 2. Scope Definition
*   **Problem Statement:** Users currently search for books by title/author individually. Workflow requires searching ISBN first, then title/author. System lacks integration with external book metadata (Open Library) to enrich catalog when books not found locally.
*   **Business Outcome:** Single unified search endpoint; seamless ISBN/text query handling with local-first Open Library fallback; rich book metadata from external source; transparent backend orchestration; zero manual ISBN entry burden.
*   **In-Scope:** Unified search endpoint (ISBN/title/author auto-detection), local DB lookup, Open Library API integration, fuzzy matching, result ranking by source/completeness, cover download on user confirmation, metadata completion tracking, deduplication at system level.
*   **Out-of-Scope:** Advanced filtering (genre hierarchy), search analytics, search result personalization, bulk import wizard.

## 3. Actors & Preconditions
*   **Primary Actors:** Authenticated user (search & add book), system administrator (monitor metadata completeness).
*   **Dependencies:** Book Catalog (SPEC-0002), Open Library API, Personal Library (SPEC-0003), Media Management (SPEC-0005).
*   **Preconditions:** User authenticated, backend has internet connectivity (graceful degradation if Open Library unavailable).

## 4. Functional Requirements

### 4.1 Search Capabilities
*   **REQ-001:** Provide unified search endpoint that accepts single text query (ISBN, title, or author).
*   **REQ-002:** Auto-detect query type: if normalized string is 10 or 13 digits → ISBN; otherwise → text query.
*   **REQ-003:** For ISBN queries: first search local DB by ISBN-13 and ISBN-10 (case-insensitive, normalized).
*   **REQ-004:** For text queries: search local DB by title (substring, case-insensitive, fuzzy) and author (substring, case-insensitive, fuzzy); return top 10 local results.
*   **REQ-005:** If local search yields results → include in response with source="LOCAL_DB" and status describing user's ownership state (see 4.2).
*   **REQ-006:** If no local results → query Open Library API as fallback.
*   **REQ-007:** Open Library search: for ISBN → use `/api/books?bibkeys=ISBN:{isbn}` endpoint; for title/author → use `/search.json?title={query}` endpoint.
*   **REQ-008:** Map Open Library response fields to OSR Book model (see OPEN_LIBRARY_ANALYSIS.md section 2).
*   **REQ-009:** Open Library results returned with source="OPEN_LIBRARY" and status="AVAILABLE_FOR_IMPORT".
*   **REQ-010:** Limit result set: max 10 results per source (local top 10, Open Library top 10).
*   **REQ-011:** Rank results by completeness: COMPLETE > INCOMPLETE; within same completeness → no guaranteed order.

### 4.2 Search Response Status Field
*   **REQ-012:** Each result includes `status` enum with values:
    - `EXISTS_IN_USER_LIBRARY` — book exists in global catalog AND in authenticated user's library
    - `EXISTS_IN_SYSTEM` — book exists in global catalog but NOT in user's library
    - `AVAILABLE_FOR_IMPORT` — book not in local DB; found in Open Library
*   **REQ-013:** If status="EXISTS_IN_USER_LIBRARY" → frontend shows "Already in your library" message (disable add button).
*   **REQ-014:** If status="EXISTS_IN_SYSTEM" → frontend shows "Add to library" button (enabled).
*   **REQ-015:** If status="AVAILABLE_FOR_IMPORT" → frontend shows "Add to system & library" button (enabled with visual indicator for external source).

### 4.3 Book Addition Flow
*   **REQ-016:** When user clicks "Add to system & library" for Open Library result:
    1. Backend creates Book in local catalog (map Open Library fields per OPEN_LIBRARY_ANALYSIS.md)
    2. Backend adds Book to user's library (SPEC-0003 flow)
    3. Return book_id and user_book_id for frontend
*   **REQ-017:** Cover handling: Open Library `cover_url` returned in search response as `coverUrl`; if import creates a new catalog book and `coverUrl` is present, backend attempts to download and persist the cover through the SPEC-0005 media flow without blocking the import if that download fails.
*   **REQ-018:** Deduplication: if multiple concurrent requests attempt to import same Open Library book (same ISBN-13), database unique constraint or serializable transaction prevents duplicate; second request receives existing book_id.
*   **REQ-019:** On successful book import from Open Library → set Book fields:
    - `metadataCompletionStatus = "COMPLETE"` or `"INCOMPLETE"` based on field availability
    - `externalSourceId = "{open_library_work_key_or_isbn}"` for audit trail
    - `createdBy = authenticated_user_id` (tracks importer)

### 4.4 Metadata Completeness
*   **REQ-020:** If Open Library response missing `pages`, `publisher`, or `language` fields → mark book `metadataCompletionStatus="INCOMPLETE"`.
*   **REQ-021:** In search response, include `metadataCompletionStatus` field; frontend may show banner: "Book info incomplete; system will enhance".
*   **REQ-022:** Admin may trigger background task to enrich incomplete books via scheduled ISBN re-lookup; deferred to Phase 2.

### 4.5 Error Handling & Resilience
*   **REQ-023:** If Open Library API unavailable (timeout, 5xx) → return local results only with warning in response: `"externalSearchFailed": true`.
*   **REQ-024:** Implement retry logic: max 3 attempts with exponential backoff (1s, 2s, 4s); timeout per request: 5s.
*   **REQ-025:** Cache Open Library results for 24h (Redis or in-memory) to reduce API load.
*   **REQ-026:** If query produces no local or external results → return empty result set with descriptive message.

### 4.6 API Endpoint
*   **REQ-027:** Provide endpoint: `POST /books/search-unified` accepting:
    ```json
    {
      "query": "string (ISBN, title, or author)",
      "limit": "integer, optional, default=10, max=100"
    }
    ```
*   **REQ-028:** Response structure:
    ```json
    {
      "results": [
        {
          "bookId": "uuid or null",
          "title": "string",
          "primaryAuthor": "string",
          "otherAuthors": ["array of strings"],
          "isbn13": "string or null",
          "isbn10": "string or null",
          "publisher": "string or null",
          "publicationDate": "YYYY-MM-DD or null",
          "pages": "integer or null",
          "language": "string (ISO 639-1)",
          "genres": ["enum array"],
          "coverUrl": "string or null (Open Library URL, not persisted)",
          "source": "enum: LOCAL_DB | OPEN_LIBRARY",
          "status": "enum: EXISTS_IN_USER_LIBRARY | EXISTS_IN_SYSTEM | AVAILABLE_FOR_IMPORT",
          "metadataCompletionStatus": "enum: COMPLETE | INCOMPLETE",
          "externalSourceId": "string or null"
        }
      ],
      "count": "integer",
      "query": "string (original query)",
      "searchedSources": ["array of: LOCAL_DB, OPEN_LIBRARY"],
      "externalSearchFailed": "boolean (true if Open Library unavailable)"
    }
    ```

## 5. Non-Functional Requirements
*   **NFR-001:** Search latency <1s for local-only queries (typical case); <3s if Open Library fallback needed.
*   **NFR-002:** Open Library API timeout: max 5s per request; retry backoff as per REQ-024.
*   **NFR-003:** Result caching: TTL 24h for Open Library responses.
*   **NFR-004:** Concurrent import deduplication: serializable transaction isolation for book creation.
*   **NFR-005:** Fuzzy matching: allow substring + accent-insensitive matching (reuse TitleAuthorNormalizer from SPEC-0002).

## 6. Business Rules

*   **RULE-001:** ISBN-13 is primary deduplication key; ISBN-10 is secondary validation only (inherited from SPEC-0002).
*   **RULE-002:** Normalized title+author is fallback dedup key if ISBN unavailable (inherited from SPEC-0002).
*   **RULE-003:** Open Library import always creates new Book entry with `createdBy = importing_user_id`; no re-use of existing books with same ISBN if user explicitly imports (integrity preserved).
*   **RULE-004:** Cover URL from Open Library is external input only; backend may persist the downloaded image through SPEC-0005 storage, but does not keep the external Open Library URL as the final cover reference.
*   **RULE-005:** If Open Library query returns multiple results for same ISBN → select first result; no manual disambiguation in MVP.
*   **RULE-006:** Genre mapping: unknown Open Library subjects → log warning, skip genre assignment (allow empty set).
*   **RULE-007:** Language code conversion: Open Library ISO 639-3 → OSR ISO 639-1 (eng→en); unknown codes default to "en".
*   **RULE-008:** Rate limiting: if Open Library requests exceed 1 req/s threshold, implement exponential backoff + in-memory circuit breaker.

## 7. Acceptance Criteria

*   **AC-001:** Search by ISBN (13 digits) → found in local DB → returns 1 result with `source="LOCAL_DB"`, `status="EXISTS_IN_SYSTEM"`.
*   **AC-002:** Search by ISBN (not in local DB) → found in Open Library → returns 1 result with `source="OPEN_LIBRARY"`, `status="AVAILABLE_FOR_IMPORT"`, includes `coverUrl`.
*   **AC-003:** Search by title "El Señor" → returns local canonical books matching the query; if no local results exist, backend falls back to Open Library results.
*   **AC-004:** Search by author "Tolkien" → returns local canonical matches; if no local matches exist, backend falls back to Open Library results ranked by completeness.
*   **AC-005:** User adds Open Library book → POST creates Book in local DB + adds to user library; returns book_id + user_book_id.
*   **AC-006:** Concurrent imports of same Open Library ISBN → only one Book entry created; second request receives existing book_id (no duplicate).
*   **AC-007:** Open Library unavailable → search returns local results only; response includes `externalSearchFailed=true`.
*   **AC-008:** Search result with incomplete metadata (missing pages) → marked `metadataCompletionStatus="INCOMPLETE"`; frontend may display banner.
*   **AC-009:** User already has book in library → search result shows `status="EXISTS_IN_USER_LIBRARY"` (disable add button on frontend).
*   **AC-010:** When an Open Library result includes `coverUrl` and import creates a new catalog book, backend attempts to persist the cover through SPEC-0005 media storage; import still succeeds if cover download/storage fails.

## 9. Open Questions / Risks / Decisions

*   **Decision Log (2026-07-07):**
	- Search is transparent to frontend; backend handles ISBN/title/author detection and source orchestration.
  - Cover URLs from Open Library are used as import-time media sources; backend attempts to persist the image through SPEC-0005 storage without blocking book creation on cover failure.
	- Concurrent imports handled by DB unique constraint + serializable transaction isolation.
	- Genre mapping: unknown subjects logged but do not block import; user/admin can complete genres post-import via SPEC-0002 book editing.
	- Open Library as MVP external source; Google Books/other APIs deferred to Phase 2.

*   **Q-001:** Should admin have endpoint to force re-enrich incomplete books from Open Library?
	- **Answer:** Yes, Phase 2 feature; deferred.

*   **Q-002:** Should we store Open Library work/edition keys in audit table for historical traceability?
	- **Answer:** Yes; `externalSourceId` field captures this.

*   **R-001:** Open Library rate limiting; implement retry + backoff.
*   **R-002:** Open Library API availability; graceful fallback to local search only.
*   **R-003:** Large result sets from title search; pagination + limit to top 10 per source.
*   **R-004:** Concurrent imports race condition; use DB-level UNIQUE constraint + serializable isolation.
*   **R-005:** Subject→Genre mapping maintenance; plan admin UI for Phase 2.
