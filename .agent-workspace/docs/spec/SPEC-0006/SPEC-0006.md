## 2. Scope Definition
*   **Problem Statement:** Users need a structured, navigable web interface with hierarchical page flow, visual library dashboard with statistics and paginated book listings, and a contextual user menu to access profile, settings, help, and library management features.
*   **Business Outcome:** Intuitive, organized user experience enabling quick access to library management features, clear navigation patterns, and discoverable self-service options (profile, settings, help).
*   **In-Scope:** 
	*   Hierarchical SPA navigation (Login → Dashboard → Library/Books/Profiles/Settings/Help).
	*   Dashboard home page with library statistics (total count, state distribution, rating average) and paginated personal library listing.
	*   Paginated book list (10 items/page) with book cards displaying cover, title, author, reading state, and rating.
	*   User context menu with options: Profile (view/edit + logout), Home, Add Book, Settings, Help.
	*   Add Book form with structured data entry.
	*   Help page with application feature documentation.
	*   Frontend authentication state persistence (30-day session remember).
	*   Search and filter capabilities on library listing.
*   **Out-of-Scope:** 
	*   Advanced recommendation UI.
	*   Social sharing features.
	*   Offline-first PWA features (deferred to Phase 3).
	*   Real-time collaborative editing.

## 3. Actors & Preconditions
*   **Primary Actors:** Authenticated end user, guest/anonymous user (pre-login).
*   **Dependencies:** React Router v6 (SPA routing), TanStack Query v5 (server state management), backend API (SPEC-0001 to SPEC-0005 endpoints).
*   **Preconditions:** 
	*   Backend API deployed and running with SPEC-0001..SPEC-0005 endpoints.
	*   Authentication token stored and retrievable from browser storage.
	*   User must be authenticated to access Dashboard, Library, Profile, Settings, Help pages.

## 4. Functional Requirements

### 4.1 Navigation & Routing
*   **REQ-001:** Implement hierarchical SPA routing with React Router v6; define routes:
	*   `/ ` or `/login` → Login page (unauthenticated users).
	*   `/dashboard` → Dashboard home with statistics and paginated library (authenticated users).
	*   `/library` → Full library view with advanced filters (authenticated users).
	*   `/profile` → User profile view/edit modal or dedicated page (authenticated users).
	*   `/add-book` → Add book form modal or dedicated page (authenticated users).
	*   `/settings` → Application settings page (authenticated users).
	*   `/help` → Help/documentation page (unauthenticated + authenticated users).
*   **REQ-002:** Implement route guards: unauthenticated users redirected to `/login` when accessing protected routes; authenticated users skip login and go directly to `/dashboard` on successful authentication.
*   **REQ-003:** Persist authentication token to browser localStorage with 30-day expiry window; auto-restore session on app reload if token valid and not expired.

### 4.2 Dashboard & Statistics
*   **REQ-004:** Dashboard displays:
	*   Library statistics section (top of page):
		*   Total count of books in personal library.
		*   State distribution: count of books per reading state (Pending, Reading, Read).
		*   Average rating of all read books (or N/A if no read books).
	*   Below statistics: paginated library listing (see 4.3).
*   **REQ-005:** Statistics computed and cached via TanStack Query; refetch on library mutations (add/remove/state-change book).

### 4.3 Library Listing & Pagination
*   **REQ-006:** Library list displays books in paginated format:
	*   10 books per page (server-side or client-side pagination via API response).
	*   Book card contains (in single-line layout if space permits, or responsive grid):
		*   Book cover image (thumbnail, ~60-80px height).
		*   Book title (truncated if >40 chars).
		*   Author name (truncated if >30 chars).
		*   Reading state badge/label (Pending, Reading, Read).
		*   Rating (stars or numeric, e.g., ⭐ 4.5 or N/A if not rated).
	*   Book card context menu/inline actions:
		*   "Change State" → modal to select next reading state.
		*   "Edit Rating" → modal to rate/review (only if state = Read).
		*   "Remove from Library" → confirm + delete.
		*   "View Details" → navigate to book detail page (or modal).
*   **REQ-007:** Implement search and filter:
	*   Text search input (searches title/author across all pages via API or client).
	*   Filter dropdown: by reading state (All, Pending, Reading, Read).
	*   Results update paginated list; reset pagination to page 1 on filter/search change.
*   **REQ-008:** Pagination controls:
	*   "Previous" / "Next" buttons or numeric page selector.
	*   Display current page and total pages.
	*   Disable "Previous" on page 1; disable "Next" on last page.

### 4.4 User Context Menu
*   **REQ-009:** Header includes user context menu (avatar or "Menu" button):
	*   Profile option → navigate to `/profile` or open profile modal.
	*   Home option → navigate to `/dashboard`.
	*   Add Book option → navigate to `/add-book` or open add-book modal.
	*   Settings option → navigate to `/settings`.
	*   Help option → navigate to `/help`.
	*   Logout option → clear session, redirect to `/login`.
*   **REQ-010:** Menu accessible from all authenticated pages; consistent placement (top-right corner or hamburger menu on mobile).

### 4.5 Profile Page
*   **REQ-011:** Profile page displays:
	*   User avatar (if available) with upload option.
	*   Display name (with edit inline or in modal).
	*   Email (read-only).
	*   Edit button → form to update display name and avatar.
	*   Logout button → confirmation dialog, then clear session + redirect to `/login`.

### 4.6 Add Book Form
*   **REQ-012:** Add Book form collects:
	*   ISBN-13 (optional; trigger catalog lookup if provided).
	*   Title (required).
	*   Author (required).
	*   Publisher (optional).
	*   Language (dropdown, default "en").
	*   On submit: call library add endpoint with selected book; on success, show confirmation and redirect to `/dashboard` or `/library`.
	*   Search existing catalog first; allow selection from results before adding to library.

### 4.7 Settings Page
*   **REQ-013:** Settings page placeholder with sections:
	*   Account Settings (future: password change, email verification).
	*   Notification Preferences (future: email notifications, reminders).
	*   Display Preferences (future: dark mode, language).
	*   Note: MVP phase includes placeholder UI only; no backend integration required yet.

### 4.8 Help Page
*   **REQ-014:** Help page displays:
	*   Overview section: "OpenShelfRating helps you catalog your personal library, track reading progress, and rate books."
	*   Feature sections with descriptions and screenshots or illustrations:
		*   Library Management: Add, search, filter books; manage reading states.
		*   Reading Lifecycle: Track reading progress (Pending → Reading → Read).
		*   Ratings & Reviews: Rate and write opinions on books you've read.
		*   Profile & Media: Upload avatar and book cover images.
		*   Account Security: Social login, email verification, password reset.
	*   Accessible to both authenticated and unauthenticated users.

### 4.9 Authentication State & Session Management
*   **REQ-015:** Implement session persistence:
	*   On successful login (SPEC-0001 endpoint), store JWT token in localStorage with expiry timestamp (current time + 30 days).
	*   On app load: check localStorage for token; if present and not expired, skip login page and restore session.
	*   On logout: remove token from localStorage, clear in-memory auth state, redirect to `/login`.
*   **REQ-016:** Login failure handling:
	*   After 10 failed attempts within 30 minutes: temporarily block login for 2 minutes.
	*   Display countdown timer to user.
	*   Clear failed-attempt counter after successful login or after 30-minute window.

## 5. Non-Functional Requirements
*   **NFR-001:** Page load time for Dashboard ≤2s (with mocked data).
*   **NFR-002:** Pagination response latency ≤500ms from API.
*   **NFR-003:** Search/filter applies within 300ms (client-side filtering or optimized API query).
*   **NFR-004:** Component re-render on state mutation should not exceed 100ms.
*   **NFR-005:** Browser localStorage token storage: secure (httpOnly if possible via proxy), validated on each protected route access.
*   **NFR-006:** Responsive design: mobile-first, breakpoints at 480px (mobile), 768px (tablet), 1024px (desktop).
*   **NFR-007:** Accessibility: WCAG 2.1 AA compliance; keyboard navigation, screen reader support on all interactive elements.

## 6. Business Rules
*   **RULE-001:** Only authenticated users can access Dashboard, Library, Profile, Add Book, Settings pages.
*   **RULE-002:** Session token expires after 30 days of inactivity or app closure; user must re-login.
*   **RULE-003:** Failed login attempts are rate-limited: 10 attempts per 30-minute window; 2-minute block on 10th failure.
*   **RULE-004:** User can only view/edit their own profile and library; admin can override (Phase 2).
*   **RULE-005:** Add Book form must first search global catalog; if book not found, allow creation of new catalog entry (subject to admin moderation, deferred Phase 2).
*   **RULE-006:** Statistics (totals, averages) cached and refreshed on any library state change (add, remove, state transition, review update).

## 7. Acceptance Criteria
*   **AC-001:** Unauthenticated user navigates to `/dashboard`; redirected to `/login` page.
*   **AC-002:** User logs in with valid credentials; JWT token stored in localStorage with 30-day expiry; redirected to `/dashboard` immediately.
*   **AC-003:** User closes app and reopens within 30 days; session restored from localStorage; dashboard displays without re-login.
*   **AC-004:** User closes app and reopens after 30 days; token expired; redirected to `/login`.
*   **AC-005:** Dashboard displays statistics: total books, state counts, average rating; values match backend aggregations.
*   **AC-006:** Library list shows 10 books per page; page 2 button disabled if ≤10 total books; "Next" disabled on last page.
*   **AC-007:** Search input filters books by title/author; pagination resets to page 1; results update within 300ms.
*   **AC-008:** Filter dropdown (by state) updates list; pagination resets; only books matching state displayed.
*   **AC-009:** User clicks book card "Change State"; modal shows available state transitions; on confirm, state updated and statistics refreshed.
*   **AC-010:** User logs in 10 times with wrong password within 30 minutes; 11th attempt shows "Account temporarily locked; try again in 2 minutes."
*   **AC-011:** User clicks "Logout" in user menu; confirmation dialog appears; on confirm, session cleared, localStorage token removed, redirected to `/login`.
*   **AC-012:** User navigates to `/profile`; sees current name, email, avatar; clicks edit, updates name, saves; changes reflected on reload.
*   **AC-013:** User navigates to `/add-book`; enters ISBN; results from catalog appear; selects book, confirms; added to library, redirected to dashboard.
*   **AC-014:** User navigates to `/help`; sees feature documentation with descriptions of library management, reading lifecycle, ratings, profile, and account security.
*   **AC-015:** User menu accessible from header on all authenticated pages; menu items (Home, Profile, Add Book, Settings, Help, Logout) all functional and navigate correctly.

## 9. Open Questions / Risks
*   **Q-001:** Should "Edit Book Details" (title, author, etc.) be allowed for books already in user's library (Phase 2)?
*   **Q-002:** Should Help page include video tutorials or only text/screenshots (MVP: text + screenshots)?
*   **Q-003:** Should book detail page (AC-007 "View Details") be a modal or full page (MVP: modal; full page Phase 2)?
*   **R-001:** localStorage security: if app is not HTTPS, tokens stored plain-text. Mitigation: deploy HTTPS-only in production.
*   **R-002:** 30-day session expiry long for security; consider rolling expiry (refresh on each API call) in Phase 2.
*   **R-003:** Search/filter performance on large libraries (10k+ books); may require backend-side pagination + advanced indexing (Phase 2).
