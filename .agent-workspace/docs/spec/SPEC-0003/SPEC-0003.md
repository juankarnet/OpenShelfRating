# Specification: SPEC-0003 - Personal Library Management

## 1. Metadata
*   **Spec ID:** SPEC-0003
*   **Version:** 0.2.0
*   **Status:** Draft
*   **Owner:** BA Agent, Architect Agent
*   **Source Milestone:** Functional Alignment
*   **Last Updated:** 2026-06-26

## 2. Scope Definition
*   **Problem Statement:** Each user needs a personal collection of books with reading state tracking independent of other users' collections.
*   **Business Outcome:** Personal library per user; reading progress visibility; foundation for reviews and recommendations.
*   **In-Scope:** Add book to library, remove from library, view library with state, search within library, list libraries (admin).
*   **Out-of-Scope:** Sharing libraries with other users, library templates, bulk import in MVP.

## 3. Actors & Preconditions
*   **Primary Actors:** Authenticated user, system administrator.
*   **Dependencies:** Book Catalog (SPEC-0002), Identity & Access (SPEC-0001).
*   **Preconditions:** User account created, book exists in global catalog.

## 4. Functional Requirements
*   **REQ-001:** Add existing book to user's library (links user + book via UserBook relation).
*   **REQ-002:** Remove book from user's library (soft delete: mark deleted_at, preserve history).
*   **REQ-003:** List user's library with pagination (default 20 books/page), filterable by reading state, excluding soft-deleted records by default.
*   **REQ-004:** Search within user's library by book title/author.
*   **REQ-005:** Query user library stats: total books, books by state (pending/reading/read).
*   **REQ-006:** Provide endpoints: POST /users/{id}/library, DELETE /users/{id}/library/{book_id}, GET /users/{id}/library, GET /users/{id}/library/stats. Listing endpoints must support optional `includeDeleted=true`.
*   **REQ-007:** Prevent duplicate entries: user cannot add same book twice to library.
*   **REQ-008:** Re-adding a previously soft-deleted book must reactivate the existing UserBook record instead of creating a new one.

## 5. Non-Functional Requirements
*   **NFR-001:** Library list retrieval: <500ms for user with 200+ books.
*   **NFR-002:** Pagination: max 100 books per page to avoid memory spike.

## 6. Business Rules
*   **RULE-001:** One user cannot have two active UserBook entries for the same book (enforced by PostgreSQL partial unique index where deleted_at IS NULL).
*   **RULE-002:** Only resource owner or admin can mutate user's library; authenticated principal identity is the source of truth.
*   **RULE-003:** Soft delete of UserBook preserves audit trail; data retained for analytics (deferred).
*   **RULE-004:** Default reading state on add-to-library: PENDING.
*   **RULE-005:** Reading state transitions beyond initial PENDING assignment are governed by SPEC-0004.

## 7. Acceptance Criteria
*   **AC-001:** User adds book to library → GET /users/{id}/library returns book with state=PENDING.
*   **AC-002:** User tries to add same book twice → second request returns 409 Conflict.
*   **AC-003:** User removes book → book is excluded from GET /users/{id}/library by default; record is returned only with `includeDeleted=true`.
*   **AC-004:** Filter library by state=READING → returns only books with that state.
*   **AC-005:** Stats endpoint returns: total=50, pending=10, reading=15, read=25.
*   **AC-006:** User removes a book and adds it again → previous record is reactivated (same user_book_id), no duplicate is created.
